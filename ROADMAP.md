# Roadmap

## Phase 1: Foundation

- Strengthen `oqs-safe` as a Rust post-quantum cryptography toolkit
- Improve documentation, examples, and release discipline
- Build developer trust through testing, security notes, and stable APIs
- Document hybrid cryptography and secure session use cases

## Phase 2: Platform

- Advance Quantum Vault as a quantum-safe KMS
- Add APIs, policy control, audit logging, and deployment guides
- Demonstrate use cases for digital assets, secure storage, and high-assurance infrastructure

## Phase 3: AI Security

- Expand agentic AI security labs
- Add memory poisoning, context manipulation, prompt injection, and tool-abuse scenarios
- Create repeatable AI red-team simulations

## Phase 4: Governance and Resilience

- Improve blast-radius simulation
- Add evidence reports
- Support replay, explainability, and mitigation tracking
- Align with enterprise AI risk and operational resilience needs

## Phase 5: Ecosystem

- Publish articles
- Create reference architecture diagrams
- Invite contributors
- Build demos
- Apply for grants
- Position the framework for enterprise, open-source, and research adoption
